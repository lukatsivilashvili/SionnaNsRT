# Sionna ns-3 to Linux TAP proof-of-chain

This example sends UDP traffic generated by an ns-3 application from an LTE UE across a Sionna RT-managed LTE hop, through the EPC PGW, over a CSMA link to a ghost node, and then to Linux through TapBridge.

Start Sionna RT from the repository root:

```bash
cd src/sionna
python3 sionna_v1_server_script.py --local-machine --frequency=2.1e9 --path-to-xml-scenario=scenarios/SionnaExampleScenario/scene.xml
```

The user prompt used `sionna_server_v1_script.py`; this tree currently contains `sionna_v1_server_script.py`.

Build and run the ns-3 example from the repository root:

```bash
./ns3 build
./ns3 run --enable-sudo "sionna-ns3-to-linux-tap --tapName=sionna-tap0 --simTime=60"
```

Do not run the ns-3 wrapper itself with `sudo`; this tree's `./ns3` script refuses to run as root. Use `--enable-sudo` so the wrapper can request privileges only for the TAP helper.

Verify from Linux:

```bash
sudo tcpdump -i sionna-tap0 -n -vv udp port 9000 -w sionna-tap0-udp9000.pcap
```

To test ns-3 routing without TapBridge, run:

```bash
./ns3 run "sionna-ns3-to-linux-tap --enableTap=false --simTime=10"
```

If `ConfigureLocal` cannot create the TAP in your environment, use manual `UseLocal` setup:

```bash
sudo ip tuntap add dev sionna-tap0 mode tap user $USER
sudo ip addr add 10.10.2.2/24 dev sionna-tap0
sudo ip link set sionna-tap0 up
./ns3 run --enable-sudo "sionna-ns3-to-linux-tap --tapMode=UseLocal --tapName=sionna-tap0 --simTime=60"
```

Useful options:

```bash
--packetSize=512
--interval=100ms
--trafficApp=OnOff
--dataRate=40960bps
--port=9000
--distance=15
--localMachine=true
--sionnaServerIp=127.0.0.1
--enablePcap=true
--enableFlowMonitor=true
```

The LTE version uses `car_1` for the UE/source node and `car_2` for the eNB node, matching the repository convention where ns-3 node IDs map through `obj<N+1>` to Sionna scene objects named `car_N+1`. The TAP is still on a separate CSMA device attached to the ghost node, not on the LTE device.
